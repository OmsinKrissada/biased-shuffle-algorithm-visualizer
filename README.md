# "Shuffle Using In-place Swaps" Visualizer

## This demo explores a pattern that emerges from a _seemingly uniform_ shuffle algorithm.

This web-based demo was made for a friend to explain how a shuffle algorithm using seemingly random swaps of elements in an array does not result in what you might expected.

Shuffle algorithm labeled `Math.random() * length` represents the following algorithm written in Python:

```py
def shuffle(array):
    length = len(array)
    for i in range(length):
        newLocation = math.floor(random.random() * length)
        temp = array[i]
        array[i] = array[newLocation]
        array[newLocation] = temp
    return array
```

and `Math.random() * (i + 1)` represents the following:

```py
def shuffle(array):
    length = len(array)
    for i in range(length):
        newLocation = math.floor(random.random() * (i + 1))
        temp = array[i]
        array[i] = array[newLocation]
        array[newLocation] = temp
    return array
```

On the surface, it might seems like both ways result in an array with an evenly distributed chance of getting items from each initial position.

Running both algorithm 10 million times (each iteration doesn't depend on the result of the previous loop) shows that while this hold true for the second algorithm, the same can't be said for the first one. The former algorithm has a **noticeable biased pattern** which depends on which side the swap loop starts from. (from the left as shown in Python above and in the demo)

This website walks you through the chance **calculation** process for each swaps that explains how the biased pattern could appear.

Link to the demo can be found on the side of this repo page.

> The chances shown on the demo website is from calculation, look into the source code for more detail.
